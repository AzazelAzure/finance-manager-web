### Fixed

- **Sync status bar when offline / after errors:** hide the bar when the browser is offline and the outbox is empty (no misleading “sync” strip); on `window.offline` mark the API unreachable immediately (`markApiReachable(false)`) from `OfflineRoot` and `SyncStatusBar` so a second offline spell still preemptively queues writes; clear a stuck **`error`** sync phase when reachability returns **ok**; add **`sync.status.offlineNoQueue`** and **`sync.status.queuedWillSync`** i18n keys; drop stale **`syncDetail`** when the sync phase returns to **`idle`**.

- **PWA offline shell on first install:** added `clientsClaim: true` to the Workbox `generateSW` config so the service worker controls the page immediately after first activation; without this, an installed PWA opened offline for the first time hit the network instead of the precached shell, causing a hard navigation failure (`ERR_ADDRESS_UNREACHABLE`). The existing `registerType: \"prompt\"` update UX is unaffected — new SW versions still enter `waiting` until the user accepts via `SwUpdateBanner`.

- **Offline transaction create (network-failure fallback):** when an allowlisted mutating request (`POST`/`PATCH`/`DELETE` transactions, upcoming, categories, tags, sources) fails due to a network error while `shouldTreatAsDisconnectedForMutations()` had not yet detected the outage, the Axios adapter now **retroactively queues** the write to the IndexedDB outbox and returns a synthetic **202** instead of propagating the error to the UI. This closes the gap where the first write after connectivity loss was always lost. `SyncStatusBar` and `SyncProgressOverlay` now reliably reflect queued state because the `fm-offline-queued` event fires on retroactive queue as well as pre-emptive queue.

- **React Query `networkMode: "always"` (offline queries + mutations):** React Query v5 defaults to `networkMode: "online"`, which **pauses** all queries and mutations when `navigator.onLine` is `false` — the `queryFn`/`mutationFn` never executes. This prevented the app's custom offline-first cache reads (`preferOfflineCaches()` → IndexedDB) and outbox writes (`shouldQueueOfflineWrite()` → Dexie) from ever running while the browser was offline. Set `networkMode: "always"` in both `queries` and `mutations` default options so React Query always invokes the function, letting the app's own offline infrastructure handle network state.

- **PWA offline after clearing site data / SW reset:** removed the extra Workbox **`navigate` + StaleWhileRevalidate** runtime route so only the generated **`NavigationRoute` → precached `index.html`** handles document loads. The duplicate SWR route could run on an empty `pages` cache and hit the network while offline, causing a hard navigation failure (**ERR_ADDRESS_UNREACHABLE**) instead of serving the precached shell. **`probeApiReachability`** now skips the cross-origin fetch when `navigator.onLine` is false and marks the API unreachable immediately.

- **Offline Quick add / ledger when creating categories, tags, or sources:** mutating `POST`/`PATCH`/`DELETE` to **`/finance/categories/`**, **`/finance/tags/`**, and **`/finance/sources/`** are now on the same offline outbox allowlist as transactions and upcoming expenses, so a **new category** (or tag/source mutation) queued before a transaction no longer fails the whole save with a non-allowlisted network error.

- **Offline / PWA transaction create “missing from list”:** `txlist:*` Dexie cache keys use `JSON.parse`, which produced numeric `current_month` / flag values; list overlay used strict string checks (`=== "1"`), so pending rows were filtered out. Filters are now coerced consistently. Offline mutating queue accepts **either** refresh or access token (drain still requires refresh to upload). Queued transaction POSTs **merge synthetic pending rows** into matching `txlist:*` cache payloads for cache-first reads.

- **Sync UX on public routes + dashboard totals after ledger saves:** **`OfflineRoot`** mount/visibility/online handlers no longer run outbox drain or seed on marketing routes (`/` etc.)—only under `/app` or in an **installed PWA**—so a session with a non-empty outbox does not open **`SyncProgressOverlay`** on the landing page. **`SyncProgressOverlay`** is limited to **standalone / installed PWA** display modes; **`SyncStatusBar`** remains on `/app` in the browser. Transaction save/delete mutations now **`invalidateQueries` with `refetchType: "all"`** for snapshot, transactions, sources, and calendar/viz keys so KPI totals refresh reliably when returning to the dashboard.

- **Docker production image:** copy `.npmrc` into the build stage **before** `npm ci` so the same `legacy-peer-deps` policy as local dev applies when building the static SPA image (Vite 8 + `vite-plugin-pwa` peer range).
